📚 Learn Asynchronous JavaScript Step by Step

1️⃣ SYNCHRONOUS VS ASYNCHRONOUS - Understanding the Difference

Synchronous: Code runs line by line, one after another
Asynchronous: Code can run in parallel, doesn't block execution

Step 1.1: Synchronous Example

console.log("Start");
console.log("Middle");
console.log("End");
// Output: Start, Middle, End (in order)

Step 1.2: Asynchronous Example with setTimeout

console.log("Start");
setTimeout(() => {
    console.log("Delayed message");
}, 2000);
console.log("End");
// Output: Start, End, Delayed message (after 2 seconds)

2️⃣ CALLBACKS - Functions as Arguments

What is a callback? A function passed as an argument to another function

Step 2.1: Basic Callback

function greet(name, callback) {
    console.log("Hello " + name);
    callback();
}

function sayGoodbye() {
    console.log("Goodbye!");
}

greet("Hamad", sayGoodbye);

Step 2.2: Callback with Parameters

function calculate(a, b, operation, callback) {
    let result = operation(a, b);
    callback(result);
}

function displayResult(result) {
    console.log("Result: " + result);
}

calculate(5, 3, (x, y) => x + y, displayResult);

3️⃣ PROMISES - Better Way to Handle Async Operations

What is a Promise? An object representing the eventual completion of an async operation

Step 3.1: Creating a Promise

let promise = new Promise((resolve, reject) => {
    setTimeout(() => {
        let success = true;
        if (success) {
            resolve("Operation successful!");
        } else {
            reject("Operation failed!");
        }
    }, 2000);
});

Step 3.2: Using Promises with then/catch

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            resolve("Data received!");
        }, 1500);
    });
}

fetchData()
    .then(result => console.log(result))
    .catch(error => console.log(error));

Step 3.3: Promise Chaining

function getUser() {
    return Promise.resolve({id: 1, name: "Hamad"});
}

function getPosts(user) {
    return Promise.resolve([
        {id: 1, title: "Post 1", userId: user.id},
        {id: 2, title: "Post 2", userId: user.id}
    ]);
}

getUser()
    .then(user => getPosts(user))
    .then(posts => console.log(posts))
    .catch(error => console.log(error));

4️⃣ ASYNC/AWAIT - Cleaner Promise Syntax

What is async/await? Modern syntax for working with promises

Step 4.1: Basic async/await

async function getData() {
    try {
        let result = await fetchData();
        console.log(result);
    } catch (error) {
        console.log(error);
    }
}

function fetchData() {
    return new Promise(resolve => {
        setTimeout(() => resolve("Data loaded!"), 1000);
    });
}

5️⃣ FETCH API - Making HTTP Requests

What is Fetch? Modern way to make network requests in JavaScript

Step 5.1: Basic GET Request

fetch('https://jsonplaceholder.typicode.com/users/1')
    .then(response => response.json())
    .then(user => console.log(user))
    .catch(error => console.log('Error:', error));

Step 5.2: Fetch with async/await

async function getUser() {
    try {
        let response = await fetch('https://jsonplaceholder.typicode.com/users/1');
        let user = await response.json();
        console.log('User:', user.name);
    } catch (error) {
        console.log('Error:', error);
    }
}

Step 5.3: POST Request

async function createPost() {
    try {
        let response = await fetch('https://jsonplaceholder.typicode.com/posts', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify({
                title: 'My New Post',
                body: 'This is the content',
                userId: 1
            })
        });
        let newPost = await response.json();
        console.log('Created post:', newPost);
    } catch (error) {
        console.log('Error:', error);
    }
}

💡 Key Takeaways

  • Synchronous: Code runs in sequence, blocks execution
  • Asynchronous: Code runs in parallel, doesn't block
  • Callbacks: Functions passed as arguments to handle async results
  • Promises: Objects representing future values (pending, fulfilled, rejected)
  • async/await: Cleaner syntax for working with promises
  • Fetch: Modern API for making HTTP requests
