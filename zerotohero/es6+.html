1️⃣ ARROW FUNCTIONS - Shorter Function Syntax

What are arrow functions? A concise way to write functions using => syntax

Step 1.1: Basic Arrow Function

// Traditional function
function add(a, b) {
    return a + b;
}

// Arrow function
const add = (a, b) => a + b;

Step 1.2: Arrow Functions with Multiple Lines

const calculate = (a, b, operation) => {
    let result;
    switch(operation) {
        case 'add': result = a + b; break;
        case 'subtract': result = a - b; break;
        case 'multiply': result = a * b; break;
        default: result = 0;
    }
    return result;
};

Step 1.3: Arrow Functions and 'this'

const person = {
    name: "Hamad",
    traditional: function() {
        setTimeout(function() {
            console.log("Traditional: " + this.name); // undefined
        }, 1000);
    },
    arrow: function() {
        setTimeout(() => {
            console.log("Arrow: " + this.name); // "Hamad"
        }, 1000);
    }
};

2️⃣ DESTRUCTURING - Extracting Values from Arrays/Objects

What is destructuring? A way to unpack values from arrays or properties from objects

Step 2.1: Array Destructuring

const colors = ["red", "green", "blue"];
const [first, second, third] = colors;

// Equivalent to:
// const first = colors[0];
// const second = colors[1];
// const third = colors[2];

Step 2.2: Object Destructuring

const user = {
    name: "Hamad",
    age: 25,
    city: "Karachi"
};

const {name, age, city} = user;
// Creates variables: name, age, city

Step 2.3: Default Values and Renaming

const person = {name: "Ali"};
const {name, age = 18, city: location = "Unknown"} = person;

// name = "Ali"
// age = 18 (default)
// location = "Unknown" (renamed from city)

3️⃣ TEMPLATE LITERALS - Enhanced String Handling

What are template literals? Strings with embedded expressions using backticks (`)

Step 3.1: Basic Template Literals

const name = "Hamad";
const age = 25;

// Traditional concatenation
const message1 = "Hello, my name is " + name + " and I am " + age + " years old.";

// Template literal
const message2 = `Hello, my name is ${name} and I am ${age} years old.`;

Step 3.2: Multi-line Strings

// Traditional multi-line string
const html1 = "
\n" +
              "  
Title
\n" +
              "  
Content
\n" +
              "
";

// Template literal multi-line
const html2 = `

  
Title

  
Content


`;

4️⃣ SPREAD & REST OPERATORS - ... (Three Dots)

Spread: Expands arrays/objects
Rest: Collects multiple values

Step 4.1: Spread with Arrays

const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

// Combine arrays
const combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]

// Copy array
const copy = [...arr1]; // [1, 2, 3]

Step 4.2: Spread with Objects

const person = {name: "Hamad", age: 25};
const address = {city: "Karachi", country: "Pakistan"};

// Combine objects
const fullInfo = {...person, ...address};

// Copy and modify
const updatedPerson = {...person, age: 26};

Step 4.3: Rest Parameters

function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}

// Can accept any number of arguments
sum(1, 2, 3);        // 6
sum(1, 2, 3, 4, 5);  // 15

5️⃣ MODULES - Organizing Code

What are modules? Reusable pieces of code in separate files

Step 5.1: Export/Import Basics

// math.js (export)
export const add = (a, b) => a + b;
export const multiply = (a, b) => a * b;

// main.js (import)
import { add, multiply } from './math.js';

Step 5.2: Default Exports

// calculator.js
export default class Calculator {
    add(a, b) { return a + b; }
    subtract(a, b) { return a - b; }
}

// main.js
import Calculator from './calculator.js';

6️⃣ MODERN ASYNC FEATURES

Step 6.1: Promises

const promise = new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve("Data loaded!");
    }, 1000);
});

promise.then(result => console.log(result));

Step 6.2: Async/Await

async function loadData() {
    try {
        const result = await fetchData();
        console.log(result);
    } catch (error) {
        console.log(error);
    }
}
